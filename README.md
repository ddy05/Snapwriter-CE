# About the Project
**Snapwriter CE** is a code and text editor for Android that emphasizes developer velocity, intelligent features, and consistent reliability, aiming to make mobile editing *less painful* and *more enjoyable* with a clear, modern interface that cuts down on the hassle and simply lets focus *snap* into place. It hosts features such as: 
- **Boilerplate code auto-completion** - e.g. for HTML
- **Context-based automatic settings** - e.g. for text files: sans serif font, word wrap ON, line numbers OFF, line highlight OFF / for code files: monospace font, word wrap OFF, line numbers ON, line highlight ON
- **Per-file viewing preference overrides** - for word wrap and line numbers (e.g. you can choose to enable line numbers for one specific .txt file, or disable word wrap for just a specific code file — and have preferences saved *instantaneously*)
- and — in the future — **Code import via OCR** with linting and auto-correct integration

Snapwriter CE is currently in development stage with an alpha release date not yet set in stone. Stay tuned! 

# Project Setup
If you would like to test Snapwriter CE, you will need:
- [Android Studio Panda 4](https://developer.android.com/studio/releases)
- [Git](https://git-scm.com/install/)
- a reliable internet connection
  
First, copy the repository's web URL to clone the project.

Snapwriter CE is developed with the intent to be safe, transparent, and free from malicious behavior.

To the best of the sole developer's knowledge, this repository does **not** contain malicious code, intentional backdoors, or hidden data exfiltration mechanisms. Snapwriter CE follows an offline-first architecture and does not require an internet connection to access its core functionality.

All dependencies are publicly listed via the source code and can be audited. Independent review and feedback is welcome via the [Issue Tracker](https://github.com/ddy05/Snapwriter-CE/issues).


# Supported Languages
Snapwriter CE can theoretically edit any file with any extension. Regardless, it currently supports rudimentary syntax highlighting for the following languages:
- HTML
- CSS
- JavaScript
- PHP
- Python
- Java
- Kotlin
- C#
- SQL
- JSON
